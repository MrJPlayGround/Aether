# License And Terms Outline

This is a practical outline for a lawyer to turn into a real agreement. It is
not legal advice.

## Agreement Structure

Use two pieces:

1. services agreement
2. software license terms

That lets you sell setup/support while keeping product ownership clear.

## Core License Position

The client receives:

- a limited
- revocable
- non-exclusive
- non-transferable

license to use the deployed Aether instance.

The client does **not** receive:

- ownership of the software
- ownership of the source code
- resale rights
- sublicensing rights
- unrestricted white-label rights

## Key Clauses To Include

### 1. Ownership

State clearly:

- you retain all right, title, and interest in the software
- the client receives only the agreed usage rights
- all core product IP remains yours

### 2. License Scope

Limit use to:

- the named client
- their internal use
- the agreed number of deployments or environments

Disallow:

- resale
- redistribution
- sublicensing
- transfer to another party

### 3. Source Code

State clearly:

- source code is not part of the deliverable unless explicitly agreed
- access to a hosted or managed instance does not transfer code ownership

### 4. Restrictions

Prohibit:

- reverse engineering
- copying the software
- creating derivative commercial products from it
- removing branding or ownership notices if applicable

### 5. White-Label Rights

If you allow branding changes, distinguish them from ownership.

State:

- branding/custom naming does not transfer software ownership
- white-label rights, if any, are limited to the written scope
- broader resale/partner rights require a separate agreement

### 6. Client Data

State clearly:

- the client owns their operational data
- you may process/store data only as needed to provide the service
- on termination, you can export or return client data in a reasonable format

### 7. Confidentiality

Cover:

- your software and methods
- the client’s trading data, notes, and operational information

### 8. Fees And Payment

State:

- setup fee terms
- monthly fee terms
- payment timing
- consequences of non-payment

### 9. Term And Termination

State:

- agreement term
- renewal terms
- termination rights
- what happens on termination

Important:

- license ends when the agreement ends unless otherwise stated
- hosted access may be suspended or terminated after non-payment or termination

### 10. Support And Service Level

Set realistic expectations:

- what support is included
- response windows
- update/maintenance expectations
- what is out of scope

### 11. No Investment Advice / No Guarantee

You need explicit language that:

- the product is decision support and operational tooling
- it is not guaranteeing performance
- it is not trade execution
- it is not financial, legal, or tax advice

### 12. Limitation Of Liability

Cap liability and exclude indirect damages where allowed.

### 13. Custom Work

If you do client-specific changes, specify:

- whether the client receives any ownership in those changes
- that reusable frameworks, methods, and platform improvements remain yours

Best default:

- custom configuration belongs to the client’s deployment
- core platform improvements stay with you

## Practical Default Position

Best default commercial stance:

- client owns data
- you own software
- client licenses use
- no repo handoff
- no resale
- no source delivery

## When To Use A Separate White-Label Agreement

Use a separate agreement when the client wants:

- resale rights
- sublicensing
- broad branding rights
- multiple downstream client deployments
- source access

That should be priced much higher than normal client access.
